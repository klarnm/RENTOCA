Slide 5 — Pipeline ML (en `main.py` y `analisis_descriptivo.py`)
- Objetivo: predecir formalidad (SUNAT) → binario: Formal (1) vs Informal (0)
- Pasos principales:
  1. Cargar CSV (`sep=';'`, `encoding='latin1'`, `low_memory=False`)
  2. Filtrar filas sin target y rellenar otros nulos con 'Desconocido'
  3. Binarizar `p1_sunat` mediante heurística por texto (buscar 'NO EST', 'PERSONA', 'JURIDICA')
  4. Convertir features a `str` y aplicar `LabelEncoder` por columna
  5. `train_test_split(test_size=0.2, random_state=42)`
  6. Entrenar `DecisionTreeClassifier(max_depth=4)`
  7. Evaluación básica: `accuracy` en test
  8. Mostrar predicciones limitadas (primeras 200 filas) para la UI

Slide 8 — Datos y consideraciones de calidad
- Encoding: `latin1` usado por problemas de acentos/caracteres
- `p1_sunat` tiene entradas con artefactos/encoding; por eso se usa heurística textual
- Dataset preprocesado: ~36.975 filas (aprox.) y ~121 columnas (original)
- Muchos valores faltantes en columnas no centrales; se rellenan con 'Desconocido' para el demo

Slide 9 — Limitaciones y riesgos
- Modelo demo: Decision Tree simple, no optimizado para producción
- Heurística de binarización puede introducir errores (encoding, nuevas categorías)
- Desequilibrio de clases puede sesgar métricas (se reporta accuracy solo como referencia)
- Sesiones in-memory → no apto para producción
- Render de la tabla de predicciones limitado a 200 filas para evitar lentitud

Slide 10 — Recomendaciones técnicas (mejoras)
- Evaluación extendida: matriz de confusión, precision/recall/F1, AUC
- Balanceo de clases: `class_weight='balanced'`, oversampling/SMOTE o undersampling
- Convertir pipeline a `sklearn.pipeline` + serializar modelo (joblib) para producción
- Paginación en UI + endpoint API para predicciones en lotes
- Persistencia de sesiones y credenciales seguras (DB / OAuth)
